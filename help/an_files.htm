При наличии у Вас прав Вам доступен Отчет "Файлы без описания" (кнопка  в меню программы, вкладка "Файлы без описания".).

В этом отчете показаны файлы всех файловых реестров программы, у которых нет описания, либо описание короче 6 символов.

Рассмотрим форму отчета:

Рис. 6.52 Отчет «Файлы без описания»: форма отчета.

(1) – Данные для отчета: в отчет выбираются файлы из выбранных файловых реестров.

По умолчанию выбраны все файловые реестры программы.

Можно выбрать какой-либо определенный файловый реестр, или несколько файловых реестров.

Для выбора одного файлового реестра кликните по его названию.

Для выбора нескольких файловых реестров кликните по первому реестру, нажмите и держите кнопку Ctrl и кликайте по остальным интересующим Вас реестрам.

Для выбора всех реестров кликните по пункту «-выбрать все-«.

В зависимости от Ваших прав у вас могут отсутствовать те или иные файловые реестры в форме выбора.

(2) – кнопка  («найти») - осуществляет поиск актов списания по указанным параметрам в фильтре (1) .

(3) – кнопка  («сбросить все фильтры») – сбрасывает все параметры фильтра (1), ищет все файлы согласно общей логике отчета.

(4) – кнопка  («печать») – производится печать отчета согласно параметрам фильтров (1).

Пример.

Найдем все файлы без описания в реестре «Файлы и документы».

Выберем этот реестр и нажмем кнопку :

 Рис. 6.53 Отчет «Файлы без описания»: результат поиска.

Рассмотрим полученные результаты (Рис. 6.40):

(1)   Файл: имя файла, является активной ссылкой, по клику по нему можно скачать файл.

(2)   Описание файла (пустое – если нет описания)

(3)   Файловый реестр – в каком файловом реестре программы находится файл. Является активной ссылкой, по клику – происходит открытие в новом окне той папки того реестра, где находится этот файл.

(4)   Папка – в какой папке находится файл. Является активной ссылкой, по клику – происходит открытие в новом окне той папки того реестра, где находится этот файл.

(5)   Дата загрузки файла

(6)   Сотрудник, загрузивший файл

(7)   Родительский документ. Данные заполнены, если этот файл относится к определенному документу программы, например, это файл определенной заявки. В этом случае указан вид, номер документа, и по клику в новом окне откроется этот документ.

(8)   Организация: поле заполнено, если файл прямо или косвенно относится к определенной организации в программе (например, файл заявки относится к заявке, которая ,в свою очередь, относится к определенной организации). В этом случае в ячейке указана организация, к которой относится файл.
В нашем примере – файл относится ко всем организациям программы, т.к. реестр «Файлы и документы» един для всех организаций. Поэтому поле не заполнено.
