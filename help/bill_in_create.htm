Создание входящего счета при наличии у Вас прав на данное действие производится с помощью кнопки "Создать входящий счет" в карте исходящего счета.

Создание входящего счета доступно из исходящего счета, у которого утверждены цены.

Рассмотрим создание входящего счета на примере исх. счета, созданного в предыдущей главе.

Для создания счета нажмем на кнопку "Создать входящий счет":

Рис. 6.15 Создание входящего счета из исходящего: кнопка "Создать входящий счет".

Нажмем кнопку "Создать входящий счет". В результате откроется окно создания входящего счета:

Рис. 6.16 Окно создания входящего счета.

Рассмотрим поля и блоки окна создания исходящего счета.

В блоке «Статус» (1) указан статус счета.

Статус счета меняется следующим образом:

  • Счет создали, но не утвердили – статус «не утвержден».
  • Счет утвердили – статус «утвержден».
  • По счету утверждена приемка, либо есть утвержденные связанные документы, но все позиции не завезены – статус «не выполнен».
  • Все позиции счета поставлены – статус «выполнен».
  • Аннулировали счет (только из статуса «не утвержден», либо произошло автоматическое аннулирование счета) – статус «аннулирован».

Блок "Контрагент" (2). В этом блоке необходимо выбрать контрагента-поставщика. Для выбора нажмите кнопку "...". Откроется окно выбора контрагента из справочника:

Рис. Окно выбора контрагента.

В окне доступен поиск контрагента по следующим полям:

  • Код
  • Полное наименование
  • ИНН
  • КПП
  • Юридический адрес

Для поиска необходимо вписать информацию в эти поля и нажать кнопку "Найти" (2).

Для выбора контрагента в счет отметьте его в столбце "Код" и нажмите кнопку "Готово" (3).

Рис. Окно создания входящего счета: контрагент выбран.

Блок «Договоры» (3).

В этом блоке необходимо выбрать договор поставщика. В блок автоматически заносится основной входящий договор из карты контрагента-поставщика.

Если нужно выбрать другой договор, нажмите кнопку "..." в блоке. Откроется окно выбора договора:

Рис. 6.17 Диалог выбора договора.

В нашем примере у контрагента введен только один договор. Если договоров несколько, то нужно отметить требуемый договор в столбце "№ договора" и нажать Готово.

После выбора договор будет подставлен в соответствующее поле окна создания счета:

В блоке «Реквизиты контрагента» (4) можно выбрать реквизиты поставщика.

По умолчанию подставляются основные реквизиты, или, если реквизиты у этого поставщика только одни, то единственные реквизиты.

 

Блок «Позиции счета» (5).

Данный блок позволяет редактировать количество и цены позиций в счете.

С помощью кнопки  можно удалить позицию из счета.

В нашем примере у позиций счета не проставлены цены. Чтобы проставить цены или отредактировать количества, нужно нажать кнопку «Редактировать позиции…».

Откроется окно редактирования позиций счета:

Рис. 6.19 Окно редактирования позиций счета.

Окно редактирования позиций состоит из трех таблиц:

  • Позиции в счете (1) – это позиции, находящиеся в данном счете
  • Доступные позиции по исх. счету (2) – это доступные позиции в исходящем счете, из которого сформирован данный счет, и которые не были включены в редактируемый счет. В нашем примере таких позиций нет.
  • Доступные услуги (3): список доступных услуг для включения в счет. Услуги включаются в счет независимо от заявок и без привязки к какой-либо определенной заявке.

В колонке «Кол-во» (4) уже заполнены количества позиций в счете. Можно их отредактировать.

В колонке «Итоговая цена» (5) необходимо указать цены на единицу каждой позиции, либо - в колонке «Всего» (6) необходимо указать стоимость данной позиции.

Колонки «Кол-во» (4), «Итоговая цена» (5) и «Всего» (6) взаимосвязаны.

При изменении колонки «Кол-во» (4) программа автоматически пересчитает колонку «Всего», подставив туда результат умножения колонки (4) и (5)

При изменении колонки «Итоговая цена» (5) программа автоматически пересчитает колонку «Всего», подставив туда результат умножения колонок (4) и (5).

При изменении колонки «Всего» (6) программа автоматически пересчитает колонку «Итоговая цена», подставив туда результат деления колонки (6) на колонку (4)

После внесения всех данных необходимо нажать кнопку «Готово». Данные будут внесены во входящий счет:

Рис. 6.21 Создание счета: цены назначены.

 

Кнопка  («создать новое поступление») (6) служит для создания поступления выделенных позиций.

Если кнопка имеет вид  - неактивна, то поступление создать невозможно по одной из причин:

  • У счета  не утверждена приемка
  • У Вас нет прав на создание поступления.

Кнопка  («редактировать количество») (7) позволяет редактировать количества выделенных позиций в счете.

Если кнопка имеет вид  - неактивна, то кол-ва позиций невозможно редактировать по одной из причин:

  • По счету есть связанные утвержденные документы
  • Счет не в статусе «не утвержден»
  • У Вас нет прав на это действие.

Кнопка  (8) – «Печать счета» - позволяет распечатать счет.

Кнопка  (9) – «Выровнять позиции с фактически полученными» - позволяет провести выравнивание позиций счета в соответствии со связанными со счетом поступлениями. Для этого необходимо отметить нужные позиции и нажать эту кнопку.

Если кнопка имеет вид  - неактивна, то выравнивание позиций произвести невозможно по одной из причин:

  • У счета не утверждена приемка
  • У Вас нет прав на выравнивание позиций.

Кнопка «Создать входящий счет» (10) позволяет создать счет с введенными данными и перейти в реестр счетов.

Кнопка «Создать входящий счет и перейти к утверждению» позволяет создать счет с введенными данными и остаться в карте счета с возможностью утвердить счет (если у Вас есть права на данное действие).

Кнопка «Закрыть форму» позволяет закрыть форму, не сохраняя изменения.

Вкладки "Поступления товара", "Оплаты" (11) содержат реестры связанных со счетом документов.

Флаги «Без автовыравнивания» и «без автоаннулирования» предназначены для исключения этого счета из автоматического выравнивания и автоматического аннулирования входящих счетов.

При наличии у Вас прав на установку этого флага, и при условии, если у счета не утверждена приемка, Вы можете установить этот флаг.

Нажмем кнопку «Создать входящий счет и перейти к утверждению» (10).

Откроется окно просмотра/редактирования созданного счета.
