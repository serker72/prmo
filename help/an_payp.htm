При наличии у Вас прав Вам доступен Отчет «План оплаты» исходящий (кнопка  в меню программы, вкладка "Исход.", "План оплаты".).

В этом отчете показаны не оплаченные, либо частично оплаченные исходящие счета по умолчанию за все время работы программы.

Счет считается не оплаченным, если:
а) по счету нет реализаций и нет ни одной оплаты;
б) по счету есть реализации, но нет ни одной оплаты;

Счет считается частично оплаченным, если:
а) по счету нет реализаций, но есть оплаты, при этом сумма оплат меньше суммы счета;
б) по счету есть реализации и есть оплаты, при этом сумма оплат меньше суммы по реализациям счета.

Рассмотрим форму отчета:

Рис. 6.32 Отчет «План оплаты»: форма отчета.

(1) – фильтр «Контрагент»: Вы можете осуществлять отбор исходящих счетов по нескольким контрагентам.

Для этого в форму поиска нужно вписать фрагмент названия искомого контрагента. Программа выдаст список вариантов названий. Выберите в списке требуемого контрагента щелчком мыши по его названию. При необходимости отобрать в отчет счета по еще одному контрагенту - снова наберите фрагмент его названия и выберите контрагента из всплывшего списка вариантов названий контрагентов.

(2) – период: позволяет отбирать для отчета исходящие счета с плановыми датами оплаты (датами оплаты по договору), попадающими в выбранный период.

(3). Фильтр "Только выполненные счета" позволяет показать только те исходящие счета, позиции по которым полностью завезены (т.е. счета в статусе "выполнен").
(4) Фильтр "Только невыполненные счета" позволяет показать только исходящие счета, позиции которых не завезены, либо завезны частично (т.е. счета в статусе "не выполнен", "утвержден", "оплачен", "не оплачен").
Одновременно может быть отмечен только один из этих двух фильтров.

(5) Фильтр "Только неоплаченные счета" позволяет показать только те исходящие счета, по которым не было ни одной оплаты (т.е. в графе "Оплачено по счету" 0 руб.).
(6) Фильтр "Только частично оплаченные счета" позволяет показать только те исходящие счета, по которым:
а) по счету нет реализаций, но есть оплаты, при этом сумма оплат меньше суммы счета;
б) по счету есть реализации и есть оплаты, при этом сумма оплат меньше суммы по реализациям счета.
Одновременно может быть отмечен только один из этих двух фильтров.

Фильтры можно сочетать, например:
Сочетание фильтров Только выполненные счета и Только неоплаченные счета выдаст исходящие счета в статусе "выполнен", по которым не было оплат.
Сочетание фильтров Только невыполненные счета и Только неоплаченные счета выдаст исходящие счета в статусах "не выполнен", либо "утвержден", по которым не было оплат.

(7) – кнопка  («найти») - осуществляет формирование отчета по указанным параметрам в фильтрах (1)-(6) .

(8) – кнопка  («сбросить все фильтры») – сбрасывает все параметры фильтров (1)-(6), формирует отчет согласно общей логике.

(9) – кнопка  («печать») – производится печать отчета согласно параметрам фильтров (1)-(6).

Пример.

Сформируем отчет по контрагенту «Инда Хаус». Для этого введем в поле «Контрагент» текст:

Инда Хаус

И нажмем кнопку :

Рис. 6.33 Отчет «План оплаты»: результат поиска по названию поставщика.

Полученные результаты (Рис. 6.33) можно сортировать по:

  • № счета в программе
  • Сумме счета
  • Статусу счета
  • Контрагенту
  • Дате оплаты по договору
  • Наличию счета в бухгалтерии

В отчете есть возможность проставлять, либо снимать ранее проставленное наличие счета в бухгалтерии. Для этого необходимо отметить требуемые счета (6) (снять галочку, если счет не в бухгалтерии, проставить – если в бухгалтерии), и нажать кнопку  для внесения изменений. Программа запросит подтверждение изменений.

Эта функция полностью аналогична функции отчета в главе «Отчет «Счета, переданные в бухгалтерию».
