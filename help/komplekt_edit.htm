Рис. 6.9 Окно редактирования заявки.

Рассмотрим блоки окна редактирования заявки.

Блок «статус» (1) – на Рис. 6.9 имеет значение «новая». У каждой заявки в данном блоке указан ее статус.

Статус заявки меняется следующим образом:

  • Заявку создали, но не утвердили – статус «Новая».
  • Заявку утвердили – статус «Утверждена». Для утверждения заявки ее должен утвердить менеджер (блок 11).
  • Сняли утверждение заявки – статус «Не утверждена».
  • Создали связанные документы по заявке, завезли не все позиции – статус «Не выполнена».
  • Завезли все позиции (полностью, либо с превышением) – статус «Выполнена».
  • Аннулировали заявку (только из статуса «не утверждена», либо произошло автоматическое аннулирование заявки) – статус «Аннулирована».

Блок позиций заявки (2):

Если заявка не утверждена, то с помощью кнопки «Добавить позиции» возможно добавить позиции к заявке.

Если заявка не утверждена, то с помощью кнопки  можно удалить позиции заявки.

В таблице позиций доступна статистика движения позиций заявки:

  • В колонке «Во вх.счетах» указано число позиций во входящих счетах
  • В колонке «Получено» указано число позиций в поступлениях.
  • В колонке «Доступно к закупке» указано число позиций, доступных для оформления по заявке входящего счета.
  • В колонке "В исх. счетах" указано число позиций в исходящий счетах
  • В колонке "Отгружено" указано число позиций в реализациях
  • В колонке "Доступно к продаже" указано число позиций, доступных для оформления по заявке исходящего счета.
  • В колонке "Итоговый результат" указано число позиций в графе "Отгружено" минус число позиций в графе "Получено".

При клике по каждому числу в этих колонках открывается окно детализации, в которым перечислены соответствующие документы товародвижения.

Кнопка  (3) – «Создать новый счет» позволяет создать новый исходящий счет по позициям заявки. Для этого необходимо отметить нужные позиции и нажать эту кнопку.

Если кнопка имеет вид  - неактивна, то счет создать невозможно по одной из причин:

  • Заявка не утверждена
  • У Вас нет прав на создание счета.

Кнопка  (4) – «Сменить наименование» позволяет сменить наименование выбранных позиций в заявке с сохранением количеств позиций.

Если кнопка имеет вид  - неактивна, то смена наименования невозможна по одной из причин:

  • У Вас недостаточно прав для этого действия
  • Заявка в статусе «Аннулирована»

Кнопка  (5) – «Редактировать количество» позволяет редактировать количество у выбранных позиций в заявке.

Если кнопка имеет вид  - неактивна, то действие невозможно по одной из причин:

  • Недостаточно прав для этого действия
  • Заявка в статусе «Аннулирована»
  • Заявка была утверждена по всем доступным ролям утверждения заявки

Кнопка (6) - "Рентабельность заявки" позволяет получить данные по рентабельности заявки.

Кнопка (7) - "Копирование заявки" позволяет провести копирование заявки и подчиненных документов между связанными организациями.

Кнопка  (8) – «Печать заявки» - позволяет распечатать заявку.

Кнопка  (9) – «Выровнять позиции с фактически полученными» - позволяет провести выравнивание позиций заявки в соответствии со связанными с заявкой поступлениями и реализациями. Для этого необходимо отметить нужные позиции и нажать эту кнопку.

Если кнопка имеет вид  - неактивна, то выравнивание позиций произвести невозможно по одной из причин:

  • Заявка не утверждена
  • У Вас нет прав на выравнивание позиций.

Блок «Примечания» (10) позволяет добавлять примечания по заявке. Количество примечаний не ограничено. Для добавления примечаний нажмите кнопку «Добавить примечания…». Откроется окно создания/редактирования примечаний:

Рис. 6.10 Окно создания/редактирования примечаний.

Впишите текст примечания и нажмите кнопку «Добавить».

Примечания можно редактировать при помощи кнопки  справа от редактируемых примечаний.

Откроется окно, аналогичное Рис. 6.10.

При наличии прав возможно удаление примечаний.

Для удаления примечаний  нажмите кнопку  справа от этого примечания.

Блок утверждения заявки (11) на Рис. 6.9 позволяет утвердить или снять утверждение заявки.

Утверждение заявки производится в программе сотрудником в роли менеджера.

Редактирование заявки в статусе «Утверждена» невозможно!

Для редактирования заявки необходимо снять утверждение!

Флаги «Без автовыравнивания» и «Без автоаннулирования» (12) предназначен для исключения этой заявки из автоматического выравнивания и аннулирования заявок (см. 6.11.3).

Важное замечание!

Флаг «без автоаннулирования» распространяется только на полностью утвержденную всеми должностными лицами заявку. Если заявка не утверждена всеми должностными лицами, то она подлежит автоаннулированию.

Кнопка «Файлы» на Рис. 6.9 открывает файловый реестр, содержащий файлы, прикрепленные к заявке:

Рис. 6.11 Фрагмент окна реестра файлов заявки.

Раздел представляет собой реестр доступных для скачивания файлов (2). В реестре указаны:

  • имена файлов,
  • размер в мегабайтах,
  • кто из сотрудников загрузил файл,
  • описание каждого файла,
  • даты загрузки.

Чтобы скачать файл, необходимо либо кликнуть по имени файла, либо нажать кнопку Получить.

В зависимости от Ваших прав, у Вас могут быть показаны блоки и кнопки:

(1) Загрузить файл. С помощью этого блока Вы можете загрузить файл в реестр. Необходимо описать файл в поле «Описание файла», и затем нажать кнопку «Выберите и загрузите файл…». Откроется окно выбора файла. После выбора файл будет загружен в раздел.

В блоке (2) – кнопки:

  - редактировать описание файла;

 - удалить файл.

Вкладка «Связанные исходящие счета» и "Связанные входящие счета" на Рис. 6.9 позволяет открыть списки счетов, выписанных по данной заявке.

Вернемся к примеру. Нашу заявку утвердили. Окно просмотра заявки приняло вид:

Рис. 6.12 Окно просмотра утвержденной заявки.

Заявка перешла в статус «Утверждена».

Поля и кнопки редактирования заявки стали неактивными, но активными стали кнопки создания счета из заявки (1) , кнопка "Рентабельность заявки" , и кнопка выравнивания позиций заявки с фактически полученными (3)  

Рассмотрим вкладки «Связанные исходящие счета» (4) и «Связанные входящие счета» (5) в заявке на Рис. 6.12.

Во вкладке «Связанные исходящие счета» (5) показан реестр всех исходящих счетов, в которые входят позиции этой заявки.

Во вкладке «Связанные входящие счета» (5) показан реестр всех входящих счетов, в которые входят позиции этой заявки.
