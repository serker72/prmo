Поступления в программе «Мобильный офис» создаются из входящего счета (раздел «Счета»).

Создание поступления при наличии у Вас прав на данное действие производится с помощью кнопки  («создать новое поступление …») в карте входящего счета.

Рассмотрим создание поступления на примере , созданного в разделе "Входящие счета" справки.

Для создания поступления необходимо отметить во входящем счете те позиции, которые необходимо включить в поступление:

Рис. 5.38 Создание поступления из входящего счета. Позиции, отмеченные для включения в поступление

На Рис. 5.38 отмечены позиции входящего счета (1). В программе можно создать поступление, включающее в себя часть позиций входящего счета. Оставшиеся позиции можно включить в другие поступления. Таким образом, по одному входящему счету может быть создано несколько поступлений.

После того, как Вы отметили позиции счета, необходимо нажать кнопку   (2). В результате откроется окно создания поступления:

Рис. 5.39 Окно создания поступления.

Рассмотрим поля и блоки окна создания поступления.

В блоке «Статус» (1) указан статус поступления.

Статус меняется следующим образом:

  • Поступление создали, но не утвердили – статус «не выполнено».
  • Поступление утвердили – статус «выполнено».
  • Аннулировали поступление (только из статуса «не выполнено») – статус «аннулирован».

Блок «Позиции поступления» .

Данный блок носит информационный характер.

В колонке «Количество» указано количество позиций в данном поступлении.

Колонка «Всего по счету» содержат количества позиции в родительском счете.

Колонка «Ранее получено» содержит количество позиций в других поступлениях родительского входящего счета. Доступна детализация по каждой цифре данной колонки. Для получения детализации необходимо щелкнуть мышью по интересующему количеству.

С помощью кнопки  можно удалить позицию из поступления.

С помощью кнопки  (3 на Рис. 5.39) возможно редактирование количества позиций поступления. Для этого необходимо выделить нужные позиции и нажать кнопку .

При наличии соответствующих прав доступна возможность превысить на 10% количество позиций по распоряжению на приемку.

Уменьшение количества позиций в поступлении

Важно! В случае уменьшения количества позиций ниже доступного программа выдаст запрос о довозе позиций.

Если довоз позиций не будет проводиться, то при утверждении поступления будет сохранено заданное число позиций, при этом количество позиций во входящем счете будет уменьшено согласно кол-ву в поступлении.

Если довоз позиций будет проводиться, то при утверждении поступления будет сохранено заданное число позиций,  количество позиций во входящем счете уменьшено не будет.

В этом случае для осуществления довоза позиций, количество которых уменьшилось, необходимо создать новое поступление из входящего счета.

При утверждении поступления без довоза позиций произойдет уменьшение суммы счета.

Если счет был оплачен (полностью, либо частично), и новая сумма счета будет меньше, чем сумма оплаты по счету, то сумма оплаты по счету будет уменьшена до значения новой суммы счета.

Освободившиеся оплаты перейдут в авансовый платеж поставщика, и будут автоматически привязаны к неоплаченным счетам поставщика.

Увеличение количества позиций в поступлении

Важно! В случае увеличения количества позиций выше максимально доступного программа  при утверждении поступления сохранит заданное число позиций, при этом количество позиций во входящем счете будет увеличено согласно кол-ву в поступлении.

При этом сумма счета возрастает. Если по счету были проведены оплаты, то программа автоматически привязывает неоплаченный остаток счета к авансу по оплатам (при его наличии).

Уменьшим количество позиции 02726. Программа выдаст подтверждение:

Рис. 5.40 Окно подтверждения довоза позиций.

Выберем «Отмена - не будет довоза».

Окно создания поступления примет вид:

Рис. 5.41 Окно создания поступления. Отредактировано количество позиций.

Кнопка «Создать поступление» (4) позволяет создать поступление с введенными данными и перейти в реестр поступлений.

Кнопка «Создать поступление и перейти к утверждению» (5) позволяет создать поступление с введенными данными и остаться в карте поступления с возможностью утвердить поступление (если у Вас есть права на данное действие).

Кнопка «Закрыть форму» (6) позволяет закрыть форму, не сохраняя изменения.

Нажмем кнопку «Создать поступление и перейти к утверждению» (5).

Откроется окно просмотра/редактирования созданного поступления.
