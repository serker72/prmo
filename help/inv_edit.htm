Создание акта при наличии у Вас прав на данное действие производится с помощью кнопки «Создать инвентаризационный акт…» в реестре актов.

Создание происходит в 2 этапа: заполнение акта и утверждение акта.

При нажатии кнопки «Создать инвентаризационный акт …» открывается окно создания акта:

Рис. 5.9.2 Окно создания инвентаризационного акта.

Рассмотрим поля и кнопки окна создания акта.

Поле «Статус» (1) у вновь создаваемого акта имеет значение «не утвержден». Щелкнув по кнопке , Вы можете отменить создание акта.

Поля «Дата инвентаризации», «Склад инвентаризации» (2,3 ) – необходимые для создания акта поля.

При выборе даты инвентаризации,склада программа автоматически сформирует таблицу позиций акта по состоянию на выбранную дату на выбранном складе.

Внимание! Невозможно создать акт с более ранней датой инвентаризации, если для данного склада уже созданы и утверждены акты с более поздней датой!

Кнопка «Добавить позиции…» (4) позволяет занести в акт позиции, которые не найдены на участке, объекте по данным программы, однако, присутствуют на нем согласно внешней (например, бухгалтерской) документации.

Кнопка  («редактировать фактическое количество») (5) позволяет редактировать фактические количества позиций акта (согласно бухгалтерской документации).

Кнопка «Печать» позволяет распечатать акт, в окне создания акта данная кнопка неактивна.

Кнопка «Создать акт » (6) позволяет создать акт с введенными данными и перейти в реестр актов.

Кнопка «Создать акт и перейти к утверждению» (7) позволяет создать акт с введенными данными и остаться в карте акта с возможностью утвердить акт (если у Вас есть права на данное действие).

Кнопка «Закрыть форму» (8) позволяет закрыть форму, не сохраняя изменения.

Пример.

Создадим акт на 21.12.2014 для склада Транзитный склад:

Рис. 5.9.3 Окно создания инвентаризационного акта: выбрана дата, склад, добавились позиции.

Если какой-либо позиции нет в наличии по данным программы, ее можно добавить, нажав кнопку «Добавить позиции». Откроется окно выбора позиций номенклатуры (аналогично заявке), в котором необходимо найти добавляемые позиции, указать их количества, и добавить их.

Позиции добавятся в конец таблицы:

Рис. 5.9.4 Окно создания инвентаризационного акта: добавлены позиции, которых нет в наличии по данным программы.

Колонки таблицы акта:

1.      Исходное количество по программе: сколько позиций по данным программы было на момент создания акта

2.      Фактическое количество: сколько позиций в наличии по данным бухгалтерской документации

3.      Исходная недостача: фактическое кол-во минус исходное кол-во по программе (больше нуля)

4.      Исходные излишки: исходное кол-во по программе – фактическое кол-во (больше нуля)

5.      Текущее кол-во по программе: текущее кол-во по программе на дату инвентаризации.

6.      В связанных поступлениях: при утверждении акта автоматически заводится поступление на недостачи, кол-во позиции в этом поступлении равно количеству недостачи

7.      В связанных реализациях: при утверждении акта автоматически заводится реализация излишков, кол-во позиции в реализации равно кол-ву излишков.

При необходимости есть возможность отредактировать фактические кол-ва товаров.

Рис. 5.9.5 Вызов редактирования фактического количества позиций.

Рис. 5.9.6 Позиции отредактированы: введены новые фактические количества.

Далее производится сохранение акта с дальнейшим утверждением.

Нажмем кнопку «Создать акт и перейти к утверждению» (9 на Рис. 5.9.2).

Откроется окно просмотра/редактирования созданного акта.

Это окно аналогично рассмотренным примерам, приведем отличающийся фрагмент внизу окна:

Рис. 5.9.7 Фрагмент окна акта инвентаризации: акт создан.

Для завершения инвентаризации необходимо утвердить заполнение акта (1), сохранить акт, и затем – утвердить коррекцию складских остатков (2).

При этом при сохранении акта с простановкой утверждения коррекции складских остатков программа выдаст запрос о создании/изменении связанных документов :

Рис. 5.9.8 Запрос о создании/изменении связанных документов при утверждении коррекции складских остатков.

Необходимо нажать «Да, внести изменения». Программа утвердит акт и создаст все связанные документы.

Рис. 5.9.9 Коррекция остатков утверждена: изменения в таблице позиций

На Рис. 5.9.9 показано, что недостачи при утверждении коррекции остатков автоматически занесены в связанное поступление (1).

По излишкам автоматически создаются и утверждаются:

·        Исходящий счет

·         Реализация

По недостачам автоматически создаются и утверждаются:

·         Входящий счет

·         Поступление

Открыть и просмотреть связанные с актом документы можно на вкладке «Связанные документы» в карте акта:

Рис. 5.9.10 Коррекция остатков утверждена: вкладка «Связанные документы».
