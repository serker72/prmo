Реализации в программе «Мобильный офис» создаются из исходящих счетов (раздел «Счета», вкладка "Исходящие счета").

Создание реализации при наличии у Вас прав на данное действие производится с помощью кнопки  («создать новую реализацию…») в карте исходящего счета.

Условия, при которых возможно создание реализации:

  • у исходящего счета должна быть утверждена отгрузка
  • каждая позиция, включаемая в реализацию, должна быть завезена согласно связанным поступлениям
    Исключение: позиции товарной группы "Услуги". Реализация услуг может быть создана без наличия связанных поступлений на услуги.
  • должно быть ненулевое нереализованное количество по каждой позиции (т.е. кол-во позиции в исходящем счете минус кол-во позиции в других связанных реализациях должно быть больше 0)

Рассмотрим создание реализации на примере исходящего счета, созданного в главах "Исходящие счета"..

Для создания реализации необходимо отметить в исходящем счете те позиции, которые необходимо включить в реализацию:

Рис. 5.38 Создание реализации из исходящего счета. Позиции, отмеченные для включения в реализацию

На Рис. 5.38 отмечены позиции исходящего счета (1). В программе можно создать реализацию, включающую в себя часть позиций исходящего счета. Оставшиеся позиции можно включить в другие реализации. Таким образом, по одному исходящему счету может быть создано несколько реализаций.

После того, как Вы отметили позиции счета, необходимо нажать кнопку   (2). В результате откроется окно создания реализации:

Рис. 5.39 Окно создания реализации.

Рассмотрим поля и блоки окна создания реализации.

В блоке «Статус» (1) указан статус реализации.

Статус меняется следующим образом:

  • Реализацию создали, но не утвердили – статус «не выполнено».
  • Реализацию утвердили – статус «выполнено».
  • Аннулировали реализацию (только из статуса «не выполнено») – статус «аннулирован».

Блок «Позиции реализации» .

Данный блок носит информационный характер.

В колонке «Количество» указано количество позиций в данном поступлении.

Колонка «Всего по счету» содержит количества позиции в родительском счете.

Колонка «Ранее получено» содержит количество позиций в других реализациях родительского поступления. Доступна детализация по каждой цифре данной колонки. Для получения детализации необходимо щелкнуть мышью по интересующему количеству.

С помощью кнопки  можно удалить позицию из реализации.

С помощью кнопки  (3 на Рис. 5.39) возможно редактирование количества позиций реализации. Для этого необходимо выделить нужные позиции и нажать кнопку . Эта операция доступна только для позиций товарной группы "Услуги". У других позиций галочка для их выделения неактивна.

При наличии соответствующих прав доступна возможность превысить на 10% количество позиций по исходящему счету.

Уменьшение количества позиций в реализации доступно только для позиций товарной группы "Услуги"

Уменьшение количества позиции в реализации при ее утверждении не влияет на количество позиций в родительском исходящем счете.

Увеличение количества позиций в реализации доступно только для позиций товарной группы "Услуги"

Увеличение количества позиции в реализации при ее утверждении не влияет на количество позиций в родительском исходящем счете.

Кнопка «Создать реализацию» (5) позволяет создать реализацию с введенными данными и перейти в реестр реализаций.

Кнопка «Создать реализацию и перейти к утверждению» (6) позволяет создать реализацию с введенными данными и остаться в карте реализации с возможностью утвердить реализацию (если у Вас есть права на данное действие).

Кнопка «Закрыть форму» (7) позволяет закрыть форму, не сохраняя изменения.

Нажмем кнопку «Создать реализацию и перейти к утверждению» (6).

Откроется окно просмотра/редактирования созданной реализации.
